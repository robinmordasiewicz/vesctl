Creates azure_vnet volterra site, you can also choose to select ingress/egress gw node.
If cloud-cred flag is not passed it will create cloud cred using env vars ARM_CLIENT_ID,ARM_TENANT_ID,ARM_SUBSCRIPTION_ID and ARM_CLIENT_SECRET

Usage:
  vesctl site azure_vnet create --name az-vnet-a [flags]

Aliases:
  create, c

Examples:
vesctl site azure_vnet create

Flags:
      --action string                              Terraform action to perform, valid values are plan, apply and none (default "none")
      --allow-all-fwd-proxy                        Enable forward proxy with this site and allow all request (only valid valid when --gw-type is ingress_egress_gw)
      --azs strings                                Comma separated list of AWS Availability Zones for the given region (default [1])
      --cloud-cred string                          Use existing cloud credential object, by default it will create the azure cloud credential
                                                   			by using the env vars ARM_CLIENT_ID, ARM_SUBSCRIPTION_ID, ARM_TENANT_ID and ARM_CLIENT_SECRET
      --disk-sizes strings                         Comma separated list of AWS Disk Sizes for all nodes (default [80])
      --existing-vnet string                       Existing vnet name, where you want to deploy the site
      --existing-vnet-rg string                    Existing vnet resource group
      --fwd-proxy-policies --allow-all-fwd-proxy   List of forward proxy policies separated by comma, if give with arg --allow-all-fwd-proxy then this will be ignored (only valid valid when --gw-type is ingress_egress_gw)
      --global-networks strings                    List of global network connectors (only valid valid when --gw-type is ingress_egress_gw)
      --gw-type string                             Gateway Type valid values are: ingress_gw, ingress_egress_gw (default "ingress_gw")
  -h, --help                                       help for create
      --inside-static-rt strings                   List of simple static route for inside network (only valid valid when --gw-type is ingress_egress_gw)
      --inside-subnet-cidrs strings                Comma separated list of Ipv4 Inside Subnet CIDRs (default [192.168.1.0/24])
      --inside-subnet-names strings                Comma separated list of Existing Inside Subnet ID's, only valid when using existing vpc
      --inside-subnet-rgs strings                  Comma separated list of Inside Subnet Resource Groups, only valid when using existing vpc. If not specified then vnet-rg will be used
      --instance-type string                       Instance type for your Volterra Nodes (default "Standard_D3_v2")
  -n, --name string                                Name of the azure_vnet site
      --network-policies strings                   List of network policies separated by comma (only valid valid when --gw-type is ingress_egress_gw)
      --nodes-per-az uint32                        Number of worker nodes per az
      --outside-static-rt strings                  List of simple static route for outside network (only valid valid when --gw-type is ingress_egress_gw)
      --outside-subnet-cidrs strings               Comma separated list of Ipv4 Outside Subnet CIDRs (default [192.168.0.0/24])
      --outside-subnet-names strings               Comma separated list of Existing Outside Subnet ID's, only valid when using existing vpc
      --outside-subnet-rgs strings                 Comma separated list of Inside Subnet Resource Groups, only valid when using existing vpc. If not specified then vnet-rg will be used
  -r, --region string                              Azure region where you want to create site (default "eastus")
      --resource-group string                      [Optional] Name of the new resource group to be created, defaults to the azure_vnet site name
      --ssh-pubkey-file string                     SSH pub key file (default: $HOME/.ssh/id_rsa.pub)
      --vnet-cidr string                           Azure Vnet IPv4 CIDR blocks, given if new vnet needs to be created (default "192.168.0.0/22")
      --vnet-name string                           [Optional] Vnet Name, if not specified it will be autogenerated
  -w, --wait                                       Wait until the terraform action is complete (default true)
