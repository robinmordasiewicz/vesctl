Creates AWS VPC volterra site, you can also choose to select ingress/egress gw node.
If cloud-cred flag is not passed it will create cloud cred using env vars AWS_ACCESS_KEY_ID AWS_SECRET_ACCESS_KEY

Usage:
  vesctl site aws_vpc create --name site-a [flags]

Aliases:
  create, c

Examples:
vesctl site aws_vpc create 

Flags:
      --action string                Terraform action to perform, valid values are plan, apply and none (default "none")
      --allow-all-fwd-proxy          Enable forward proxy with this site and allow all request (only valid valid when --gw-type is ingress_egress_gw)
      --azs strings                  Comma separated list of AWS Availability Zones for the given region (default [us-east-2a])
      --cloud-cred string            Use existing cloud credential object, by default it will create the aws cloud credential
                                     by using the env vars AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY
      --disk-sizes strings           Comma separated list of AWS Disk Sizes for all nodes (default [80])
      --fwd-proxy-policies strings   List of forward proxy policies separated by comma, if give with arg "--allow-all-fwd-proxy" then this will be ignored
                                     			(only valid valid when --gw-type is ingress_egress_gw)
      --global-networks strings      List of global network connectors (only valid valid when --gw-type is ingress_egress_gw)
      --gw-type string               Gateway Type valid values are: ingress_gw, ingress_egress_gw (default "ingress_gw")
  -h, --help                         help for create
      --inside-static-rt strings     List of simple static route for inside network (only valid valid when --gw-type is ingress_egress_gw)
      --inside-subnet-ids strings    Comma separated list of Existing Inside Subnet ID's, only valid when using existing vpc
      --inside-subnets strings       Comma separated list of Inside Subnet CIDRs (default [192.168.1.0/24])
      --instance-type string         Instance type for your Volterra Nodes (default "t3.xlarge")
  -n, --name string                  Name of the aws_vpc site
      --network-policies strings     List of network policies separated by comma (only valid valid when --gw-type is ingress_egress_gw)
      --nodes-per-az uint32          Number of worker nodes per az
      --outside-static-rt strings    List of simple static route for outside network (only valid valid when --gw-type is ingress_egress_gw)
      --outside-subnet-ids strings   Comma separated list of Existing Outside Subnet ID's, only valid when using existing vpc
      --outside-subnets strings      Comma separated list of Outside Subnet CIDR's (default [192.168.0.0/24])
  -r, --region string                AWS region where you want to create site, optionally you can set it through env var AWS_DEFAULT_REGION
      --ssh-pubkey-file string       SSH pub key file (default: $HOME/.ssh/id_rsa.pub)
      --vpc-cidr string              VPC Primary IPv4 CIDR blocks, given if new vpc needs to be created (default "192.168.0.0/22")
      --vpc-id string                Existing vpc id, where you want to deploy the site
      --vpc-tag string               [Optional] VPC Name tag, if not specified it will be autogenerated
  -w, --wait                         Wait until the terraform action is complete (default true)
