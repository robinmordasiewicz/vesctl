Build a Kubernetes secret manifest with blindfold-encrypted data.

This command creates a Kubernetes Secret resource with the encrypted
secret data, ready to be applied to a cluster managed by F5 XC.

Usage:
  vesctl request secrets build-blindfold-bundle [flags]

Examples:
  # Build a secret bundle
  f5xc request secrets build-blindfold-bundle --name my-secret --data secret.txt

  # Build with custom namespace
  f5xc request secrets build-blindfold-bundle --name my-secret --namespace my-ns --data secret.txt

Flags:
      --data string        File containing secret data to encrypt
  -h, --help               help for build-blindfold-bundle
      --name string        Name of the Kubernetes secret
      --namespace string   Kubernetes namespace (default "default")
      --outfile string     Output file for the bundle

Global Flags:
  -a, --cacert string         Server CA cert file path
  -c, --cert string           Client cert file path
      --config string         A configuration file to use for API gateway URL and credentials (default "/home/ubuntu/.vesconfig")
      --hardwareKey           Use yubikey for TLS connection
  -k, --key string            Client key file path
      --outfmt string         Output format for command
  -o, --output string         Output dir for command (default "./")
      --p12-bundle string     Client P12 bundle (key+cert) file path. Any password for this file should be in environment variable VES_P12_PASSWORD
  -u, --server-urls strings   API endpoint URL (default [http://localhost:8001])
      --show-curl             Emit requests from program in CURL format
      --timeout int           Timeout (in seconds) for command to finish (default 5)
